## 一、项目概述

### 1.1 项目背景

本项目是一个低成本 AI 聚合 API 平台，为用户提供统一的 AI 模型调用入口。用户可以通过网页聊天或 API Key 方式调用多个 AI 模型，系统按 token 扣费。

### 1.2 后端定位

后端是整个平台的核心，负责：
- 用户身份认证与权限管理
- 余额管理与计费扣费
- API Key 管理与安全校验
- AI 请求转发与协议兼容
- 调用日志与数据统计
- 管理后台 API

### 1.3 核心技术栈

| 类别 | 技术选型 | 说明 |
|------|----------|------|
| 语言 | Java 21 | LTS 版本 |
| 框架 | Spring Boot 3.x | 主流企业级框架 |
| 安全 | Spring Security + JWT | 认证鉴权 |
| ORM | MyBatis Plus | 高效 CRUD |
| 数据库 | MySQL 8.0 | 主数据存储 |
| 缓存 | Redis | 限流/缓存 |
| AI 网关 | New API / One API | 模型转发 |

### 2.2 核心表结构

#### 2.2.1 用户表 (user)

```sql
CREATE TABLE user (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    username    VARCHAR(50)  NOT NULL COMMENT '用户名',
    email       VARCHAR(100) COMMENT '邮箱',
    phone       VARCHAR(30) COMMENT '手机号',
    password    VARCHAR(255) NOT NULL COMMENT '密码(bcrypt加密)',
    avatar      VARCHAR(255) COMMENT '头像URL',
    role        VARCHAR(20)  DEFAULT 'USER' COMMENT '角色: USER/ADMIN',
    status      TINYINT      DEFAULT 1 COMMENT '状态: 1正常 0禁用',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

#### 2.2.2 钱包表 (wallet)

```sql
CREATE TABLE wallet (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT NOT NULL COMMENT '用户ID',
    balance         BIGINT DEFAULT 0 COMMENT '余额(credits)',
    total_recharge  BIGINT DEFAULT 0 COMMENT '累计充值',
    total_used      BIGINT DEFAULT 0 COMMENT '累计消耗',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='钱包表';
```

> **设计说明**：余额使用整数 credits，避免小数精度问题。1元 = 10000 credits。

#### 2.2.3 钱包流水表 (wallet_log)

```sql
CREATE TABLE wallet_log (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id         BIGINT       NOT NULL COMMENT '用户ID',
    type            VARCHAR(30)  NOT NULL COMMENT '类型: RECHARGE/CONSUME/ADMIN_ADD/ADMIN_DEDUCT/REFUND',
    amount          BIGINT       NOT NULL COMMENT '变动金额(正负)',
    before_balance  BIGINT       NOT NULL COMMENT '变动前余额',
    after_balance   BIGINT       NOT NULL COMMENT '变动后余额',
    remark          VARCHAR(255) COMMENT '备注',
    created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='钱包流水表';
```

#### 2.2.4 API Key 表 (api_key)

```sql
CREATE TABLE api_key (
    id           BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id      BIGINT       NOT NULL COMMENT '用户ID',
    name         VARCHAR(100) COMMENT 'Key名称',
    api_key      VARCHAR(255) NOT NULL COMMENT 'API Key(加密存储)',
    key_prefix   VARCHAR(50) COMMENT 'Key前缀(用于展示)',
    status       TINYINT      DEFAULT 1 COMMENT '状态: 1正常 0禁用',
    total_calls  BIGINT DEFAULT 0 COMMENT '累计调用次数',
    total_cost   BIGINT DEFAULT 0 COMMENT '累计消耗',
    last_used_at DATETIME COMMENT '最后使用时间',
    created_at   DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_api_key (api_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='API Key表';
```

> **安全建议**：`api_key` 存储 SHA-256 哈希值，创建时返回明文一次。

## 五、安全设计

### 5.1 认证鉴权

1. **JWT Token**
   - 有效时间：24小时
   - 存储位置：Redis（支持主动失效）
   - 刷新策略：临近过期时刷新

2. **API Key 认证**
   - 存储：SHA-256 哈希值
   - 前缀展示：保留前12位
   - 限流：默认 60次/分钟

### 5.2 安全策略

| 策略 | 说明 |
|------|------|
| 密码加密 | BCrypt |
| SQL注入 | MyBatis 参数绑定 |
| XSS | 输入过滤 + 输出转义 |
| CSRF | JWT + SameSite Cookie |
| 限流 | Redis + 滑动窗口 |
| 敏感配置 | 环境变量 |

### 5.3 风控规则

| 规则 | 动作 |
|------|------|
| 余额不足 | 禁止调用，返回 402 |
| 用户禁用 | 禁止调用，返回 403 |
| Key禁用 | 禁止调用，返回 403 |
| 请求超限 | 限流，返回 429 |
| 异常IP | 标记 + 人工审核 |

## 六、计费设计

### 6.1 计费公式

```
总费用 = 输入费用 + 输出费用
输入费用 = prompt_tokens / 1000 × input_price
输出费用 = completion_tokens / 1000 × output_price
```

### 6.2 扣费时机

1. **流式调用**：首字返回后扣费
2. **非流式调用**：完整响应后扣费
3. **失败调用**：根据实际情况决定是否扣费

### 6.3 幂等性保证

- 使用 Redis 分布式锁
- 扣费操作加事务
- 记录扣费流水

|------|----------|

### A. 响应码规范

| 响应码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 402 | 余额不足 |
| 429 | 请求过于频繁 |

### B. 错误码定义

```java
public enum ErrorCode {
    // 通用
    SUCCESS(200, "操作成功"),
    PARAM_ERROR(400, "参数错误"),
    UNAUTHORIZED(401, "未登录"),
    FORBIDDEN(403, "无权限"),
    
    // 业务
    USER_NOT_FOUND(1001, "用户不存在"),
    USER_DISABLED(1002, "用户已被禁用"),
    PASSWORD_ERROR(1003, "密码错误"),
    USERNAME_EXISTS(1004, "用户名已存在"),
    
    // 钱包
    BALANCE_NOT_ENOUGH(2001, "余额不足"),
    
    // API Key
    API_KEY_INVALID(3001, "API Key无效"),
    API_KEY_DISABLED(3002, "API Key已被禁用"),
    
    // AI
    MODEL_NOT_FOUND(4001, "模型不存在"),
    MODEL_DISABLED(4002, "模型已停用"),
    AI_GATEWAY_ERROR(4003, "AI服务调用失败");
}
```
